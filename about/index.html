About Blog

知识管理包含了知识总结以及知识归纳整理分类，重温知识的过程是让记忆再一次深刻的过程。本人从一开始使用博客，MSN空间，新浪博客，CNBlog，到变成Evernote的重度用户，再到发现Evernote不能支持Markdown转而投向马克飞象，最终的目的还是为了让花过的时间，踩过的坑，走过的弯路有迹可循，避免将来的某一刻只能凭印象去回忆这些知识。本着好记性不如烂笔头的宗旨，打算在折腾记忆尚未磨灭之际记录一下走过的路和踩过的坑……

Evernote 第一个用的很不开心的地方在于运行实在是越来越慢。虽然电脑硬件在不断的升级，但一款笔记软件变得臃肿不堪，每次使用卡顿都很明显也不算什么太好的用户体验。第二个问题是，随着用户的增多，印象笔记也渐渐从过去的极客范变成平民范，对更新的技术不怎么感冒。比如始终不支持 Markdown 语法就很让人尴尬。我现在写点东西都是用 Markdown 格式，用印象笔记就意味着放弃 Markdown，这是让人无法接受的。有人说用马克飞象可以让 Evernote 支持 Markdown，这样却又不得不继续引入新软件，添加更多的麻烦。第三个问题是，运营压力导致的免费用户功能不断缩水。正常来说，付费用户应是在免费用户基础上给出增值服务，而不是为了让用户付费把以前承诺免费的东西直接收费。去年末的时候，印象笔记突然宣布，限制免费用户以后只能在两台设备上登录使用。我想大部分用户应该和我一样至少是三台设备吧，公司电脑、家里电脑和手机端，结果用起来非常尴尬。有人说可以付费啊，这就又引出一个问题，决定权不在自己手里。现在标准账户年费是98元是不贵，但万一以后涨成980元呢？好多年的积累，不是那么容易转走的，到时候或者被挟制付费，或者花大功夫转移系统，麻烦不说还很可能会造成数据丢失。

对用户数据的承诺，在商业时代其实真不用当回事儿。Google 当年关闭 Google Reader 的时候，网上很多人请愿请求 Google 不要关闭，因为里面有太多的用户数据了。但即使是不作恶的Google，还是只给了大家导出的选项。导出为 xml 格式貌似给了用户数据，但对于绝大多数非程序员的普通网友来说，等于这些年在 Reader 上积累的数据都丢了。

怎样避免这种情况呢？最好的方法是不依赖任何一种专有服务，诸如 Evernote、为知笔记之类，而是把所有的数据掌握在自己手里；同时采用通用数据格式，任何时候想切换成本几乎为零，而不是用某软件私有格式，导致积累越多切换成本越大。采用这种方法的好处是，任何一家公司的倒闭、服务的关停自己都不会收到影响，进而拥有了持久性，可以实现大时间周期的积累。

另外专有软件的服务还有一个问题就是很难再利用，大部分操作只能在软件内部实现，和其他服务通讯的时候必须导入导出，一来繁琐，二来会导致多个副本，维护起来也很费事儿。比如想把以前积累的文档同步到 Github 及 Gitbook 上，印象笔记也好为知笔记也罢，没有一个可以提供类似功能。但如果采用最通用的数据格式，就可以避免这类问题。

最终我选择了Atom+Hexo+Github作为自己最终的笔记系统，所有内容在本地Markdown文件中编辑好以后，无需调整就可以直接发布到 Github Pages上面生成静态博客，甚至同步在 Gitbook 上生成电子书。写文章写多了，就会发现很多时候，一些复杂的问题很难用一篇文章说清楚。这时候用系列文章写起来会更方便一些。用 Atom 维护 Markdown 文档的方式，可以直接把文件同步到 Gitbook 上，生成电子书。这样就可以有更大的发挥空间了。阳志平先生谈到读书的时候说，学习任意一个领域，你可以一上来就思考怎样写一本这个领域的书，因为抽象知识的最佳结构是树形结构，同时兼容广度与深度，而书最符合知识的树形结构。Atom+Hexo+Github的搭档堪称完美。

而图片的问题通过Atom插件markdown-image-insert可以完美解决，直接复制粘贴会通过剪贴板获取到图片，并且保存在与文档名称一致的文件夹中。

笔记作为我们知识的最基本构成，一个开放的系统、能大周期持续发展的系统、不依赖与任何一家企业或服务的系统，才是真正健壮的系统。基于这种考虑，我最终选择并推荐找一款好的文本编辑软件，来维护 Markdown 格式作为知识管理工具。文本编辑软件推荐 Atom 和 Sublime Text，装上相应插件后功能可以非常强大。

从一窍不通到搭建完整的博客，安装各种插件，折腾的过程带给我的绝不止一个好看的博客，还有一种思维方式，甚至内隐知识的进化，同时感谢开放互联网上分享知识的各路大神。
